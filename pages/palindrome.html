Check whether the given number is palindrome or not

//palindrome.java

import java.util.Scanner;

public class PalindromeCheck {
    public static boolean isPalindrome(int num) {
        int original = num;
        int reversed = 0;

        while (num > 0) {
            int digit = num % 10;
            reversed = reversed * 10 + digit;
            num /= 10;
        }

        return original == reversed;
    }

    public static void main(String[] args) {
        int testNumber = 12321; // Test case
        if (isPalindrome(testNumber)) {
            System.out.println(testNumber + " is a palindrome.");
        } else {
            System.out.println(testNumber + " is not a palindrome.");
        }
    }
}

Test Cases

Test Case 1: Palindrome Number
Input: 12321
Explanation:
The number 12321 reads the same backward as forward.
When reversed, it remains 12321.
Expected Output: "12321 is a palindrome."

Test Case 2: Non-Palindrome Number
Input: 12345
Explanation:
The number 12345 does not read the same backward; its reverse is 54321.
Since 12345 is not equal to 54321, it is not a palindrome.
Expected Output: "12345 is not a palindrome."

Test Case 3: Single-Digit Number
Input: 7
Explanation:
Any single-digit number is considered a palindrome because it reads the same in either direction.
The number 7 remains 7 when reversed.
Expected Output: "7 is a palindrome."

Test Case 4: Negative Number
Input: -121
Explanation:
The number -121 is not typically considered a palindrome because the negative sign makes it asymmetrical.
Reversed, it becomes 121-, which is not the same as -121.
Expected Output: "-121 is not a palindrome."
