//pair.java

import java.util.HashSet;

public class PairWithSum {
    public static boolean hasPairWithSum(int[] arr, int targetSum) {
        HashSet<Integer> seen = new HashSet<>();
        for (int num : arr) {
            if (seen.contains(targetSum - num)) {
                return true;
            }
            seen.add(num);
        }
        return false;
    }

    public static void main(String[] args) {
        // Test cases
        System.out.println(hasPairWithSum(new int[]{1, 4, 45, 6, 10, -8}, 16));  // True (6 + 10)
        System.out.println(hasPairWithSum(new int[]{1, 2, 4, 3}, 5));           // True (2 + 3)
        System.out.println(hasPairWithSum(new int[]{5, 7, 9, 12}, 17));         // True (5 + 12)
        System.out.println(hasPairWithSum(new int[]{8, 15, 7, 20}, 14));        // False
    }
}

bool hasPairWithSum(int arr[], int n, int target_sum) {
    bool seen[1000] = { false };
    for (int i = 0; i < n; i++) {
        if (target_sum - arr[i] >= 0 && seen[target_sum - arr[i]]) {

Algorithm to Check if Pair with Given Sum Exists:

1.Initialize an empty set seen.
2.Iterate through each element num in the array:
    1.Calculate the complement (i.e., target_sum - num).
    2.Check if the complement exists in the set seen:
        -If yes, return true (a pair exists that sums up to the target).
        -If no, add the current element num to the set seen.
3.If the loop ends without finding a pair, return false.

Detailed Steps:

1.Start by defining an empty set seen to store numbers you’ve already traversed.
2.For each element in the array:
    -Calculate the complement: complement = target_sum - num.
    -If complement exists in the set seen, this means there’s a pair of numbers whose sum is target_sum. Return true immediately.
    -Otherwise, add the current element num to the set seen.
3.If no such pair is found after checking all elements, return false.
            
Key Idea:

The program leverages a **hashing technique** (using a set) to store the numbers it has already seen. This allows it to check if the "complement" (i.e., `target_sum - current_element`) exists, in constant time.

Step-by-step Working:

1. **Initial Setup:**
   - The program initializes an empty set (or `HashSet` in Java and a boolean array in C).
   - This set will be used to store numbers as we traverse through the array.

2. **Iteration Over Array:**
   - The program starts looping over the array. For each number, it checks if there is another number already seen that, when added to the current number, gives the `target_sum`.

3. **Complement Calculation:**
   - For each number `num` in the array, the complement is calculated as `target_sum - num`.
   - **Why do we do this?**  
     Let’s say the target sum is 16 and the current number is `6`. To make the sum `16`, the other number needed is `10`. So the complement is `10`.
   
4. **Checking the Set (Hashing Logic):**
   - Now, the program checks whether the complement (`target_sum - num`) is present in the set.
   - If it is present, that means there’s another number in the array that can form the required pair with the current number. The program then returns `True`.
   - If the complement is not found, it adds the current number to the set and continues the loop.

5. **If No Pair is Found:**
   - If the loop completes and no pair is found that adds up to the `target_sum`, the function returns `False`.

Example Walkthrough:
Let’s say we have the array `[1, 4, 45, 6, 10, -8]` and the target sum is `16`.

1. Start with an empty set: `seen = {}`.
   
2. **First element**: `1`
   - Complement = `16 - 1 = 15`
   - `15` is not in the set `seen`, so add `1` to the set.
   - `seen = {1}`
   
3. **Second element**: `4`
   - Complement = `16 - 4 = 12`
   - `12` is not in the set `seen`, so add `4` to the set.
   - `seen = {1, 4}`
   
4. **Third element**: `45`
   - Complement = `16 - 45 = -29`
   - `-29` is not in the set `seen`, so add `45` to the set.
   - `seen = {1, 4, 45}`
   
5. **Fourth element**: `6`
   - Complement = `16 - 6 = 10`
   - `10` is not in the set `seen`, so add `6` to the set.
   - `seen = {1, 4, 6, 45}`
   
6. **Fifth element**: `10`
   - Complement = `16 - 10 = 6`
   - **Now 6 is found in the set `seen`**. This means `6 + 10 = 16`.
   - Return `True` because we found the pair (6, 10) that sums to 16.

Efficiency:
- **Time Complexity**: O(n)
  - The program only loops through the array once, making it efficient with a time complexity of O(n), where `n` is the number of elements in the array.
  - Checking and adding elements to a set (or HashSet) takes O(1) time on average.
  
- **Space Complexity**: O(n)
  - The set will store up to `n` numbers in the worst case (if no pair is found), so the space complexity is O(n).

Why Hashing is Efficient:
- **Hashing** (using a set) allows us to store and retrieve elements in constant time, which is why the algorithm performs much better than a brute-force approach that would involve nested loops and have O(n²) time complexity.
